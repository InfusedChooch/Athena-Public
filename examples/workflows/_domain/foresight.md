# /foresight — SOTA Predictive Positioning

> **Philosophy**: Proactively position the system against emerging capabilities rather than reacting to environment updates after they land.

---

## Execution Loop

### Phase 1: Foresight Scanning
1. Run a web search targeting recent discussions or updates in developer agent ecosystems:
   ```bash
   # Example queries:
   # "Claude Code update" "MCP tool multiplayer" "Maestro worktree" "OpenClaw AI agent"
   ```
2. Scan the Exocortex (`smart_search`) for any logged observations or case studies related to agent architectures.

### Phase 2: Capability Gap Audit
1. Compare emerging findings with active parameters in `AGENTS.md` and the framework modules under `.framework/v8.2-stable/`.
2. Evaluate areas:
   *   **Execution Isolation**: Are we using physical containers/worktrees for subagents? (See `ARC-409`).
   *   **Model Adapters**: Are we prepared for the latest SOTA reasoning models?
   *   **Unified Interfaces**: Are we leveraging standardized APIs (like MCP)?

### Phase 3: Adaptive Positioning
1. If a capability gap is detected, **do not build complex code immediately**. 
2. Proactively **position** Athena by:
   *   Scaffolding interface stubs or workflows under `.agent/workflows/_domain/`.
   *   Creating placeholder schemas in `.agent/config/` or `CAPS.json`.
   *   Filing a new internal project (Phase 1) in `PROJECTS.md` to track full integration.

---

## When to Use

*   When the user asks to analyze future trends or position for future IDE releases.
*   During quarterly codebase audits.
*   When a new major agent framework, model, or protocol (e.g., MCP) is announced.
